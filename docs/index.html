Open Source Figma + MCP

SPFR Figma Design Pipeline

Inspect designs, plan changes, execute safe mutations, sync tokens, and generate code through a local MCP workflow.

Read the README Publishing Guide Security

One-Line MCP Install

npx -y -p @spfr/figma-design-pipeline spfr-figma-design-pipeline-install --client all

Registers the MCP server for Claude, Gemini, and Codex, installs the bundled skill, packages the Claude Desktop bundle, and deploys the local Figma plugin manifest.

Figma Plugin

The Figma-side executor can be published in the Figma Community so users can find and install it directly in Figma.

It only connects to a local bridge on 127.0.0.1. Read-only inspection still works without the plugin.

Context-Efficient Skill

The bundled skill is intentionally thin. It starts with inspection and loads only the workflow resource needed for the task: inspect, mutate, tokens, or codegen.

Install Targets

  • --client claude for Claude Code
  • --client claude-desktop for the Claude Desktop bundle
  • --client gemini for Gemini CLI and bundled skill
  • --client codex for Codex CLI config

Security Model

  • No API tokens are committed in the repo.
  • Tokens are passed through client config or desktop bundle user config.
  • The plugin talks only to a localhost bridge.
  • Logs and issues should always redact tokens before sharing.

Release Checklist

  1. Run npm run check, npm test, and npm pack.
  2. Review SECURITY.md and the release-hardening checklist.
  3. Publish the npm package.
  4. Publish the Figma plugin listing in the Figma Community.
  5. Enable GitHub Pages from the docs/ directory.
